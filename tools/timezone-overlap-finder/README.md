Find practical meeting times for distributed teams by comparing each participant's local working hours.

## How it works
- Pick a date and meeting duration.
- Add participants with IANA timezones (for example `America/New_York` or `Europe/London`).
- Set each person's local working hours.
- The tool scans the selected day in 30-minute steps and lists valid overlap windows.

## Tips
- Use specific city-based timezone names to account for daylight saving time.
- Keep work windows realistic (for example 09:00 to 17:00).
- If no overlap appears, lower duration or widen one or two schedules.
